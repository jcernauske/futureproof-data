### 4. Scale estimate

Two sizing models:

- **Narrow override** — 1 row per school needing an override (i.e. `unitid` with a single proposed replacement 4-digit CIP).
- **Wide override** — 1 row per `(unitid, candidate 4-digit CIP)` pair, i.e. the full cross-product against the family's sibling CIPs. This is the upper bound if the override table holds mix-weighted allocations rather than a single hard replacement.

| CIP Family | Label | Schools ONLY General | Sibling 4-digit CIPs | Narrow Override Rows | Wide Override Rows (× 4-dig siblings) |
|---|---|---:|---:|---:|---:|

### Verdict

Across **all** 2-digit families with an `XX.01` general code, **9,197** distinct (school × family) combinations report ONLY the general 4-digit code and therefore would need an override row in the narrow model. That puts this squarely in **5,000-row territory** — a narrow override is a hand-maintainable table; a wide cross-product override balloons quickly and is likely better computed on-the-fly.
