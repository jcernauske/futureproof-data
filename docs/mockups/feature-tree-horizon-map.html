Tree-as-Horizon-Map

The chat column from feature-tree-as-map.md is unchanged. This board stacks all 9 required states top-to-bottom for review.

Spec answers, in summary
  1. Q1 — Vertical lane stacking. Three rows inside the col-span-5. The horizontal axis within each lane is the chip row (left-to-right).
  2. Q2 — Chip is a vertical card (title on top, badges below, optional unlock footer). Title gets the most vertical real estate and truncates at 32 chars with ellipsis.
  3. Q3 — Toggle lives in the column header, top-right. Single column-level switch — not repeated per lane. The horizon-anchor sub-line surfaces filter status when active.
  4. Q4 — "+N more" is a chip-shaped placeholder at row end. Click expands the lane vertically in place. No scroll, no nav. Replaced with "Show fewer" when expanded.
  5. Q5 — Empty-lane: dashed-border container, muted icon, font-body text-small text-text-muted, two-line copy (title + plain-English why).
  6. Q6 — Relatedness color bar on chip's left edge, 3px wide. Primary-Short = accent-thrive with subtle glow; Primary-Long = accent-info; Supplemental = text-muted.
  7. Q7 — Stat-delta badge: uppercase stat code (ERN/GRW/HMN/RES/ROI), signed integer, stat-tinted background at 12% opacity. Convention from pentagon overlay holds: stat codes in UI affordances are fine; voice contract bans them in Gemma's prose.
  8. Q8 — Experience badge: tiny pill, accent-caution for "mid-career", accent-alert for "senior", uppercase, micro-sized. Renders only when tier is mid or senior — a 7.6% / 0.2% slice signal.
  9. Q9 — Selected chip: 1px accent-thrive ring + soft glow. Siblings dim only within the selected chip's lane (other lanes stay full opacity).
  10. Q10 — Branch-flash keyframe reads great at chip scale: 1 → 1.06 → 1, accent-info glow, 600ms. Chip border picks up accent-info mid-pulse for extra read. Stagger 200ms on multi-match preserved.
