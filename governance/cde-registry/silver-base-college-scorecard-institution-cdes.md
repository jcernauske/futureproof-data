## State_Abbr — Detailed Decision Record

The task prompt flagged `state_abbr` for spot-check with a specific consideration: "Consider whether state_abbr should be CDE (it feeds Gold RPP join)." After reviewing the Gold architecture:

**Finding:** `state_abbr` on `base.college_scorecard_institution` does **not** feed the Gold RPP join.

**Evidence:**
1. **BEA RPP table shape** (`governance/models/silver-base-bea-rpp-physical.md` lines 20–22, 62): the Silver RPP table has `state_fips` as primary key and `state_abbr` as a UNIQUE derived column. RPP CDE set includes `state_fips`, `state_abbr`, `rpp_all_items`, `purchasing_power_multiplier`.
2. **Gold RPP consumer pattern** (`docs/specs/gold-regional-price-parities.md`, `docs/specs/mcp-bea-rpp.md`): the MCP tool `get_regional_price_parity(state)` and `compare_purchasing_power(state_a, state_b)` take a student-supplied state, not an institution-derived state. The purchasing-power adjustment is a function of **where the student plans to work**, not where the school is physically located.
3. **No direct `institution.state_abbr` → `bea_rpp.state_abbr` join** exists in any Gold spec or MCP tool signature. The program-to-career flow runs through `program_career_paths` (school + major → career outcomes) with a separately-supplied destination state for the RPP overlay.
4. **Bronze precedent** (`governance/cde-registry/raw-ingest-college-scorecard-institution-cdes.md` row 19): "Not a join key here (BEA RPP joins on `state_fips`, not `stabbr`; the two join happens at the Gold layer through a student-selected state, not an institution-to-RPP join on `stabbr`)." This reasoning holds at Silver unchanged.

**Conclusion:** `state_abbr` remains **non-CDE at Silver**. It is a display/filter field on the institution, consistent with Bronze. The aligned RPP-side `state_abbr` IS a CDE (per the BEA RPP contract) because it is a UNIQUE join key on that table — but that criticality lives on `base.bea_rpp`, not on `base.college_scorecard_institution`.

**Trip-wire for future re-evaluation:** If a spec lands that adds a direct join `college_scorecard_institution.state_abbr ⋈ bea_rpp.state_abbr` (for example, an "adjust institution COA by local cost of living" feature showing "$22,800 COA in Ohio = $18,500 in California terms"), re-run this agent and promote `state_abbr` to CDE at that time.
