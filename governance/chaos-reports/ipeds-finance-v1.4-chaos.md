### v1.1 escalation — system-office filter

**The post-filter top-25 audit found 9 administrative entities surviving the filter** (all with `instruction_expenses ≥ $1M`). Per the v1.1 chaos amendment, this triggers escalation: the filter is under-specified.

The entity classes are:
1. **Community College District Offices** with multi-million-dollar instructional spend (LA CCD, Rancho Santiago CCD, Alamo CCD).
2. **State University System Offices** with mid-million instructional spend (SUNY-System, U Mass-Central, MN State Colleges System).
3. **Spanish-language Central Offices** above the numeric floor (Inter American University of Puerto Rico-Central Office).
4. **For-profit Administrative Offices on F3** with multi-million-dollar instructional spend (Chamberlain-Administrative, DeVry-Administrative).

The common feature: every leaked entity has `total_fte_enrollment IS NULL` despite reporting multi-million-dollar instructional spend. The AND-clause does not use FTE.

#### Recommendations

**R1 (PRIMARY).** Strengthen the §6 AND-clause numeric proxy by adding an FTE-NULL OR FTE-low disjunct. Replace:

```sql
AND (
  instruction_expenses IS NULL
  OR instruction_expenses < 1000000.0
)
```

with:

```sql
AND (
  instruction_expenses IS NULL
  OR instruction_expenses < 1000000.0
  OR total_fte_enrollment IS NULL
  OR total_fte_enrollment < 50
)
```

This catches the 9 leaks (all FTE NULL) while remaining belt-and-suspenders against the false-positive direction — Direction (i) cases 1–4 still survive because they have positive FTE in the synthetic fixtures (a real teaching institution's `total_fte_enrollment` is rarely NULL or < 50). Adversarial counter-test: a real teaching institution with admin-pattern name AND `instruction_expenses ≥ $1M` AND FTE NULL would be excluded — adversarial-audit candidate is rare (a sub-50-FTE seminary or specialty school whose name happens to contain "Office" or "System"). The extension is conservative.

**Caveat: requires re-running Direction (i) against the strengthened AND-clause to confirm zero false-positive regressions.** Recommended as a v1.4.1 chaos pass after the patch lands.

**R2 (SECONDARY).** Consider raising the `instruction_expenses` numeric floor from $1M to ~$3M for system offices specifically. The audit data shows admin entities clustered in the $1.7M–$6.8M range; legitimate small teaching institutions cluster well under $1M. A $3M floor with FTE-NULL guard would catch the leaks without binding on small teaching schools (which mostly have positive FTE anyway). Less precise than R1; offered only as an alternative if FTE-based extension is rejected.

**R3 (DEFER).** Add `%administrative office%` as a 9th name pattern. This catches `Chamberlain University-Administrative Office`, `DeVry University-Administrative Office`, `Carrington College-Administrative Office`, and `San Joaquin Valley College-Central Administrative Office` — currently caught by `%central office%` only when the literal phrase appears. The `%administrative office%` clause is targeted, conservative (no English-language teaching institution legitimately has the literal phrase "Administrative Office" in its legal name), and complements R1.

### Residual-risk classes (cannot be caught by 8-pattern filter alone)

The 8-pattern set is bounded. These admin-entity classes leak through the name-pattern half of the AND-clause:

| Class | Example | Why filter cannot catch |
|---|---|---|
| Foreign-language admin offices | `Conférence des Présidents` (French) | No English admin keywords; filter is English+Spanish-only |
| Spanish-word-order admin offices | `Office Central de Gestión` | `%central office%` literal-phrase miss; only catches if `% office` end-of-name happens to apply |
| Pure-numeric-proxy admin entities | `Mountain Region Educational Authority` | No name signal — only the numeric proxy can catch, and only if R1/R2 strengthen the proxy |
| Nontraditional admin naming | `University Administrative Hub` | `%hub%` and `%administrative%` not in current pattern set |

These are **acceptable residuals** in v1.4 if R1/R2 land — the FTE-NULL extension to the numeric proxy catches the pure-numeric-proxy class without adding name patterns. The remaining true residuals (foreign-language ESL admin offices that happen to have positive FTE) are an extreme corner and are documented as known unmitigated risk.
