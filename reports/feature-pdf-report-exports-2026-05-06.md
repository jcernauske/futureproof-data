# Feature: PDF Report Exports for Guidance Counselor Conversations

**Status:** COMPLETE

## What Shipped

Two server-side PDF exports turn FutureProof's data into a printed take-away a high-school student leaves with after a 90-second guidance-counselor conversation:

1. **My Build PDF** — 2-page advisory document for a single school + major, downloadable from `/my-build`. Page 1: profile strip + verdict line + pentagon + 5-stat micro-table + cost & ROI strip (4-year cost / modeled debt / year-1 earnings / debt-to-earnings %) + 5-row career risk profile with risk-level chips. Page 2: top-6 suggested skills with printable Coursework/Clubs/Internship blanks + 3-audience questions (Ask the college / Ask your parents / Ask yourself) + glossary + sources line.
2. **Comparison PDF** — 1-page side-by-side for 2-3 schools at the same major, downloadable from CompareView. Mini-pentagons + stats-at-a-glance table with leading-cell highlight + cost & ROI block + risk-profile strip + "Where each school pulls ahead" autogenerated lines. No Gemma prose, no questions section.

The PDF translates every RPG metaphor used in-app into neutral advisory language — `boss → career risk`, `WIN/DRAW/LOSE → Low/Moderate/Elevated/High`, `gauntlet → career risk profile`, `build → plan`. This is the one register-by-surface RPG-exception in the voice guide; print readers (counselors at desks, parents at kitchen tables) need the credibility of straight advisory language.

## Key Decisions

1. **ReportLab Platypus** for rendering, not Playwright (already a dep) or WeasyPrint. Pure Python, deterministic byte output, no Chromium spawn surface.
2. **Bytes through memory, never disk.** `Response(content=bytes, media_type="application/pdf")` per the `wrapped.py` precedent. PII (student name, profile name) is verified to never write to disk by patching `Path.write_*` and `open()` writers in `test_no_pii_written_to_disk`.
3. **Comparison PDF: 2-3 builds, same 4-digit CIP family.** 4 schools either truncates names or shrinks numbers below 8pt; cross-major comparisons are incoherent (you can't compare ROI when the major differs). Cross-major triples bounce on the backend with 400 + helpful copy.
4. **Translate ALL RPG language to advisory language in the PDF.** Non-negotiable, enforced by the `RPG_TERMS_FORBIDDEN_IN_PDF` regex test against the rendered PDF text. The PDF voice is NOT the in-app voice; it's the one register-by-surface exception.
5. **Single scoped Gemma call** for the 3-audience question set. Per `feedback_scoped_llm_contexts.md`, the prompt receives only the build context Gemma needs (school, major, top-2 risks, top-2 strengths) — not the full build dump. Falls back to static copy on any failure with `gemma_path` recording the fallback path; every code path emits exactly one `logs/gemma.jsonl` record.
6. **Cap of 5 questions per audience, floor of 1.** Two of the 5 in "Ask the college" are static-mandatory; Gemma adds 0-3 more. Static fallbacks guarantee no audience block ships empty.
7. **5th "Insufficient data" risk-level chip.** Italic Roman, sentence-case, neutral gray. Renders when a boss `raw_score` is None — explicitly NOT a default-to-High (the missing-data-is-not-zero rule).
8. **Conditional data-coverage caveat** below the page-1 profile strip when `Build.career.match_quality != "full"`. Italic Nunito 7.5pt INK_MUTED, asymmetric spacing. Distinct from the suppressed CIP-substitution caveat per `feedback_no_substitution_caveat.md`.
9. **No QR code.** Target route `/build/:build_id` doesn't exist in `App.tsx` (the live build is state-driven, not URL-driven). Cut for hackathon; deferred to a follow-up spec along with the share-link route work.
10. **Two complementary frozensets for forbidden vocabulary.** `RPG_TERMS_FORBIDDEN_IN_PDF` is asserted against the full rendered PDF text. `FORBIDDEN_IN_GEMMA_OUTPUT` is the superset (adds stat abbreviations) applied only to Gemma's question text. The PDF chrome (pentagon, micro-table, glossary) deliberately renders the abbreviations, so they cannot be in the full-text frozenset.

## Deviations from Spec

1. **Risk-one-liner anchor fields.** §3.11.2 named `ai_exposure_percentile`, `bls_growth_pct`, `onet_burnout_top_driver`, `earnings_75th_pct` — none on `CareerOutcome` today. Implementation reads the closest existing fields (`adoption_percentile`, `growth_category` (categorical), first item from `burnout_drivers`, `earnings_1yr_p75`) and falls back to a level-only template when the named anchor is None. Worth a follow-up spec to plumb a numeric BLS growth field.
2. **Frontend cross-major guard** uses `major_text` string equality, not 4-digit CIP family — `CompareBuild` doesn't carry `cipcode`. Backend hard-validates the CIP family and returns 400 with helpful copy.
3. **Sources citation copy** is intentionally generic ("Sources: BLS OOH · College Scorecard · O*NET · Karpathy AI Exposure · BEA RPP. Powered by Gemma 4."); per-build data-version stamps not plumbed through.
